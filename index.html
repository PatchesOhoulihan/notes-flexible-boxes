Responsive Web Design unterstützt die flexible Natur des Web

Webweisender Artikel von Ethan Marcotte check here

Marcotte beschreibt in seinem Artikel drei Techniken, mit denen man seiner Meinung nach mediengerechte Webseiten bauen sollte.

  1. prozentbasierte, flexible Gridlayouts(Fluidgrids)

  2. flexible Bilder

  3. Media Queries

Vereint ermöglichen diese drei Techniken, dass eine Website auf die Umgebung reagiert und sich z.B der Größe des Browserfensters, oder genau genommen der Größe des Viewports, anpasst.

Die grundidee des responsive Webdesign ist es, ein und denselben HTML-Quelltext per Media Query mit unterschiedlichen CSS auszuliefern, sodass sie sich den unterschiedlichsten Umgebungen anpassen.

Responsive vs. Adaptive Layouts

Marcotte bevorzugt in seinem Artikel ein flexibles prozentbasiertes Raster. Aber nicht alle benutzten dieses, und so bürgerten sich im allgemeinen Sprachgebrauch die Unterscheidung zwischen responsive und adaptiven Layouts

  • Ein responsive Layout ist stufenlos und enthält alle drei von Marcotte propagierten Komponenten, also prozentbasiertes Grid, flexible Medien und Media Queries.

  • Ein adaptives Layout verzichtet auf das schwieriger umzusetzende Fluid Grid und liefert stattdessen per Media Query unterschiedliche feste Layoutbreiten aus.

Natürlich wurden darauf hin diskussionen geführt was jetzt responsiv ist.

Daraufhin meldete sich Jeffrey Zeldmann zu Wort, der die Meinung vertrat das responsive Webdesign nicht an eine bestimmte Technik gekoppelt werden sollte. siehe hier.

Die Quintessenz ist, dass es egal ist ob man es nun responsive oder adaptive nennt, gemeint ist letztlich alles, was Webseiten die Flexibilität gibt, um auf unterschiedlichsten Geräten eien gute Figur zu machen.

Moderne Webseiten erstellen

Bei der Erstellung des HTML, CSS und Javascript für moderne Webseiten gibt es zwei grundsätzliche verschiedene Herangehensweisen:

  1. Desktop first beginnen mit dem komplexesten Layout, zuweist auf Computer Monitor, und reduziert dieses dann für kleinere Bildschirme.

  2. Mobile first beginnt mit der kleinsten Variante und fügt für größere Bildschirme nach und nach Dinge hinzu.

Das wichtigste verfahren ist jedoch Content first. Wirklich wichtig ist der Inhalt. Gestaltung ohne Inhalt ist nicht mehr als eine Dekoration. Der Content der Seite ist der Grund warum jemand diese besucht. Jeffrey Zeldman hat diese simple Tatsache so ausgedrückt: Websites are simply delivery systems for content.

Semantische Strukturelemente in HTML5

In HTML5 gibt es neue Elemente, mit denen man die Struktur einer Seite auszeichnen kann. Gemeint sind damit Dinge wie Kopfbereich, Navigation, Inhaltsbereich, Sidebar und Fußbereich.

Übersicht Semantische Elemente HTML 5
HTML-Element Kurzbeschreibung
header Kopfbereiche, zumeist direkt hinter dem body-tag zumeist für logos, slogan und/oder Navigation(auch das semantische Nav). Kann mehrfach auf der Seite vorkommen.
nav Navigationsblöcke, diese tag ist für die wichtigsten Navigationsblöcke der Seite reserviert und nicht für eine einfache link sammelung verwendet werden.
main Hauptinhalt einer Seite, markiert den Hauptinhalt einer Seite, darf auf jeder Seite nur ein einziges mal vorkommen, und darf nicht in einem anderen Semantischen Tag stehen(nav, footer, aside)
section Einteilung in inhaltliche Abschnitte, geht es nur um die Gruppierung von Elementen nur um die Gestalutng, ist ein div eine bessere Wahl. Bespielweise lässt sich das main mit mehreren sections unterteilen, wie getting started und darauf further reading. Können ineinander verschachtelt werden
article in sich geschlossene inhaltliche Blöcke, ein Artikel sollte für sich alleine stehen können. Können ineinander verschachtelt werden
aside zusätzliche Informationen, die den umgebenden Text erläutern oder nur marginal mit ihm in Verbindung stehen. Nimmt man diesen Text aus dem Dokument heraus, sollte der Haupt Inhalt immer noch verständlich sein. Z.B Werbung.
footer Fußbereich, befinden sich zumeist ergänzende Informationen. Ist nicht nur für die ganze Seite erlaubt, sondern kann auch in Sections und Artikeln verwendet werden. Der Footer bezieht sich immer auf das Element in dem er sich befindet.

Article vs Section

Für die richtige verwendung sollte sich folgendes Beispiel vor augen gehalten werden.

  • Eine Zeitung besteht aus Abschnitten für Nachrichten, Sport und Börse etc. Jeder dieser Abschnitte ist eine section

  • Innerhalb eines Abschnitt gibt es Artikel. Ein Artikel ist eine in sich geschlossene Komposition.

Semantisches HTML gibt den Inhalten eine Bedeutung.

Denn noch lassen sich verschiedene Elemente durch CSS gleich aufstylen das es für den Betrachter keinen Unterschied macht. Beispielsweise können <h1> und ein <div> gleich ausehen

Für den normalen Anwender der Seite ist die Semantik zumeist uninteressant. Jedoch hilft die Semantik bei bei drei großen Gruppen:

  1. Suchmaschinen

  2. Screenreadern

  3. Entwicklern

Layoutbereiche wurden vor HTML5 traditionell mit <div> erstellt, auch heutzutage ist das nicht falsch jedoch ergibt sich daraus ein Problem. Denn die <div> elemente sind semantisch neutral und haben keinerlei Bedeutung.

Durch die semantischen Elemente ergeben sich neue Möglichkeiten.

  • Eine Suchmaschine könnte mit den neuen Elementen z.B Suchbegriffe im Hauptinhaltsbereich main höher bewerten als in den anderen Bereichen wie nav und footer

  • Ein Screenreader könnte eine Funktion zum lesen des Hauptinhalts anbieten, mit dem der Benutzer direkt zum Inhaltsbereich oder zur Navigation springen könnte

  • Ein Browser könnte eine Funktion zum Lesen des Hauptinhalts anbieten. Auf Knopfdruck würden dann alle anderen Breiche ausgeblendet werden.

ARIA Landmark Roles in HTML5

ARIA ist die Abkürzung für Accessible Rich Internet Applications und Teil einer Technischen Spezifikation der Web Accessibility Initiative(WAI), die sich um die Zugänglichkeit des Webs sorgen.

Dazu werden die HTML-Elementen um das Attribute role= erweitert, das einen fest definierten Wert erhält.

ARIA-Rolle

Beschreibung

banner

Diese Rolle kennzeichnet einen Bereich der Webseite, der site-relevante Informationen wie den Site-title, ein Logo und einen Untertitel oder einen Slogan enthält. In der Regel ist das ein Kopfbereich, der relative weit oben auf der Seite steht.

navigation

Kennzeichnet Haupt- und Unterverzeichnis zur Navigation, könnte aber auch für andere Elemente wie Inhaltsverzeichnisse, Sitemap oder ein Suchformular eingesetzt werden.

main

Kennzeichnet den Hauptinhalt einer Seite

article

ein Inhaltlicher Abschnitt auf einer Seite, der in sich geschlossen ist und unabhängig von der Seite funktioniert.

complementary

Ein solcher Bereich enthält ergänzende Informationen zum Hauptinhalt, die aber auch unabhängig von der Seite funktioniert.

note

für beiläufig eingeschobene Inhalte, die ohne den Hauptinhalt nicht unbedingt verständlich sein müssen

contentinfo

Informationen über den Inhalt der Webseite wie z.B Autor, Copyright, Fußnoten und dergleichen mehr.

Semantische Strukturen im Alltag.

Bei einer bestehenden Website gibt es keinen Grund, irgendwas am Quelltext zu ändern. Die Site wird auch mit altem HTML nach wie vor einwandfrei funktionieren.

Semantische Elemente per CSS stylen

In den allermeisten Browsern ist das styling der neuen Elemente unproblematisch. Vorsichtshalber sollte man den semantischen Elementen jedoch die Regel display:block; geben., da Browser unbekannte Elemente erst einmal als Inline-Elemente ansehen.

Auf der anderen Seite kann das auch mit der normalize.css erreicht werden.

Ansonsten ist es auch möglich diese mit einem HTML5shiv zu stylen. Manchmal wird der HTML5Shiv übrigens auch als HTML5Shim bezeichnet.
